This specification defines an API that allows web applications to talk to servers and devices that have their own protocols incompatible with those available on the web.

This is a work in progress. All contributions welcome.

{{TCPSocket}} interface 

        [Exposed=(Window,Worker), SecureContext]
        interface TCPSocket {
          constructor(DOMString remoteAddress, unsigned short remotePort,
              optional TCPSocketOptions options = {});

          readonly attribute Promise<TCPSocketConnection> connection;
          readonly attribute Promise<undefined> closed;

          Promise<undefined> close();
        };

Instances of {{TCPSocket}} are created with the internal slots described in the following table:

Internal slot Initial value Description (non-normative)
[[\readable]] `null` A {{ReadableStream}} that receives data from the socket
[[\writable]] `null` A {{WritableStream}} that transmits data to the socket
[[\connectionPromise]] `null` A {{Promise}} used to wait for the connection to be established. Corresponds to the {{TCPSocket/connection}} member.
[[\closedPromise]] `null` A {{Promise}} used to wait for the socket to close or error. Corresponds to the {{TCPSocket/closed}} member.

constructor() method

The {{TCPSocket/constructor()}} steps are:
  1. If [=this=]'s [=relevant global object=]'s [=associated Document=] is not [=allowed to use=] the [=policy-controlled feature=] named "[=policy-controlled feature/direct-sockets=]", throw a "{{NotAllowedError}}" {{DOMException}} and return `null`.
  2. If only one of |options|["{{TCPSocketOptions/localAddress}}"] or |options|["{{TCPSocketOptions/localPort}}"] is specified, throw a {{TypeError}} and return `null`.
  3. If |options|["{{TCPSocketOptions/keepAlive}}"] is specified and |options|["{{TCPSocketOptions/keepAliveDelay}}"] is not, throw a {{TypeError}} and return `null`.
  4. If |options|["{{TCPSocketOptions/keepAlive}}"] is not specified and |options|["{{TCPSocketOptions/keepAliveDelay}}"] is, throw a {{TypeError}} and return `null`.
  5. If |options|["{{TCPSocketOptions/keepAliveDelay}}"] is less than one second, throw a {{TypeError}} and return `null`.
  6. If |options|["{{TCPSocketOptions/sendBufferSize}}"] is equal to 0, throw a {{TypeError}} and return `null`.
  7. If |options|["{{TCPSocketOptions/receiveBufferSize}}"] is equal to 0, throw a {{TypeError}} and return `null`.
  8. Perform the following steps [=in parallel=].
    1. Invoke the operating system to open the TCP socket using the connection parameters (or their defaults) specified in |options|.
    2. If this fails for any reason, [=reject=] the {{TCPSocket/[[connectionPromise]]}} with a "{{NetworkError}}" {{DOMException}}, detach {{TCPSocket/[[closedPromise]]}} and abort these steps.
    3. On success initialize {{TCPSocket/[[readable]]}} and {{TCPSocket/[[writable]]}} and resolve {{TCPSocket/[[connectionPromise]]}} populating all necessary options.
  9. Return [=this=].

TCPSocketOptions dictionary

            dictionary TCPSocketOptions {
              unsigned long sendBufferSize;
              unsigned long receiveBufferSize;

              DOMString localAddress;
              unsigned short localPort;

              boolean noDelay;
              boolean keepAlive;
              unsigned long keepAliveDelay;
            };
sendBufferSize member
The requested send buffer size, in bytes.
receiveBufferSize member
The requested receive buffer size, in bytes.
localAddress member
The IP address (or hostname) on the local host that the socket should be bound to.
localPort member
The port that the socket should be bound to. When omitted, or 0, any port can be used.
noDelay member
Enables the `TCP_NODELAY` option, disabling Nagle's algorithm.
keepAlive member
Enables/disables TCP Keep-Alive by setting `SO_KEEPALIVE` option on the socket.
keepAliveDelay member
Specifies the Keep-Alive ping delay in milliseconds. Must be specified only if {{TCPSocketOptions/keepAlive}} is set to `true`.

{{TCPSocket/[[readable]]}} attribute (internal)

The {{TCPSocket/[[readable]]}} creation steps are:
  1. Let |stream:ReadableStream| be a [=new=] {{ReadableStream}}.
  2. Let |pullAlgorithm| be the following steps [=in parallel=]:
    1. Invoke the operating system to read a tcp packet from the socket, placing the result in the [=byte sequence=] |bytes|.
    2. [=Queue a global task=] on the [=relevant global object=] of [=this=] using the [=serial port task source=] to run the following steps:
      1. If EOF packet was received, run the following steps:
        1. Invoke [=ReadableStream/close=] on [=this=].{{TCPSocket/[[readable]]}}.
        2. Invoke the steps to [=handle closing the readable stream=].
      2. If no errors were encountered run the following steps:
        1. Let |buffer| be a [=new=] {{ArrayBuffer}} created from |bytes|.
        2. Let |chunk| be a [=new=] {{Uint8Array}} view over |buffer|, who's length is the length of |bytes|.
        3. Invoke [=ReadableStream/enqueue=] on [=this=].{{TCPSocket/[[readable]]}} with |chunk|.
      3. If an operating system error was encountered, invoke [=ReadableStream/error=] on [=this=].{{TCPSocket/[[readable]]}} with a "{{NetworkError}}" {{DOMException}} and invoke the steps to [=handle closing the readable stream=].
      4. If |the socket was disconnected|, run the following steps:
        1. Reject [=this=].{{TCPSocket/[[closedPromise]]}}.
        2. Invoke [=ReadableStream/error=] on [=this=].{{TCPSocket/[[readable]]}} with a "{{NetworkError}}" {{DOMException}}.
        3. Invoke the steps to [=handle closing the readable stream=].
  3. Let |cancelAlgorithm| be the following steps (in parallel):
    1. Invoke the operating system to close the socket.
    2. Invoke the steps to [=handle closing the readable stream=].
  4. [=ReadableStream/Set up=] |stream| with pullAlgorithm set to |pullAlgorithm|, cancelAlgorithm set to |cancelAlgorithm|, highWaterMark set to 0.
  5. Set [=this=].{{TCPSocket/[[readable]]}} to |stream|.
To handle closing the readable stream perform the following steps:
  1. If [=this=].{{TCPSocket/[[writable]]}} is already errored:
    • If [=this=].{{TCPSocket/[[readable]]}} has been errored, [=reject=] [=this=].{{TCPSocket/[[closedPromise]]}}.
    • If [=this=].{{TCPSocket/[[readable]]}} has been closed, [=resolve=] [=this=].{{TCPSocket/[[closedPromise]]}} with `undefined`.

{{TCPSocket/[[writable]]}} attribute (internal)

The {{TCPSocket/[[writable]]}} creation steps are:
  1. Let |stream:WritableStream| be a [=new=] {{WritableStream}}.
  2. Let |signal:AbortSignal| be |stream|'s [=WritableStream/signal=].
  3. Let |writeAlgorithm| be the following steps, given |chunk|:
    1. Let |promise:Promise| be [=a new promise=].
    2. Assert: |signal| is not [=AbortSignal/aborted=].
    3. If |chunk| cannot be [=converted to an IDL value=] of type {{BufferSource}}, reject |promise| with a {{TypeError}} and return |promise|. Otherwise, save the result of the conversion in |source:BufferSource|.
    5. [=In parallel=], run the following steps:
      1. Invoke the operating system to write |bytes| to the socket.
        The operating system may return from this operation once |bytes| has been queued for transmission rather than after it has been transmitted.
      2. [=Queue a global task=] on the [=relevant global object=] of [=this=] using the [=serial port task source=] to run the following steps:
        1. If the chunk was successfully written, [=resolve=] |promise| with `undefined`.
          [[STREAMS]] specifies that |writeAlgorithm| will only be invoked after the {{Promise}} returned by a previous invocation of this algorithm has resolved.
        2. If an operating system error was encountered, [=reject=] |promise| with an "{{NetworkError}}" {{DOMException}}.
        3. If |the socket was disconnected|, run the following steps:
          1. [=Reject=] |promise| with a "{{NetworkError}}" {{DOMException}}.
          2. Invoke the steps to [=handle closing the writable stream=].
        4. If |signal| is [=AbortSignal/aborted=], [=reject=] |promise| with |signal|'s [=AbortSignal/abort reason=].
    6. Return |promise|.
  4. Let |abortAlgorithm| be the following steps:
    1. Let |promise| be [=a new promise=].
    2. Run the following steps [=in parallel=]:
      1. Invoke the operating system to shutdown the socket for writing.
      2. Invoke the steps to [=handle closing the writable stream=].
      3. [=Resolve=] |promise| with `undefined`.
    3. Return |promise|.
  5. Let |closeAlgorithm| be the following steps:
    1. Let |promise| be [=a new promise=].
    2. Run the following steps [=in parallel=].
      1. Invoke the operating system to shutdown the socket for writing.
      2. Invoke the steps to [=handle closing the writable stream=].
      3. If |signal| is [=AbortSignal/aborted=], [=reject=] |promise| with |signal|'s [=AbortSignal/abort reason=].
      4. Otherwise, [=resolve=] |promise| with `undefined`.
    3. Return |promise|.
  6. [=WritableStream/Set up=] |stream| with writeAlgorithm set to |writeAlgorithm|, abortAlgorithm set to |abortAlgorithm|, closeAlgorithm set to |closeAlgorithm|, highWaterMark set to 1.
  7. [=AbortSignal/Add=] the following abort steps to |signal|:
    1. Cause any invocation of the operating system to write to the socket to return as soon as possible no matter how much data has been written.
  8. Set [=this=].{{TCPSocket/[[writable]]}} to |stream|.
To handle closing the writable stream perform the following steps:
  1. If [=this=].{{TCPSocket/[[readable]]}} is already closed or errored:
    • [=Reject=] [=this=].{{TCPSocket/[[closedPromise]]}}.

connection attribute

The {{TCPSocket/connection}} is an attribute containing all information about the socket connection. When called, returns the {{TCPSocket/[[connectionPromise]]}}.

closed attribute

The {{TCPSocket/closed}} is an attribute that keeps track of the socket state. It gets resolved if the socket is closed gracefully (i.e. by the user) or rejected in case of an error. When called, returns the {{TCPSocket/[[closedPromise]]}}.

close() method

The {{TCPSocket/close()}} method steps are:
  1. If [=this=].{{TCPSocket/[[connectionPromise]]}} is rejected or not yet resolved, [=reject=] with "{{InvalidStateError}}" {{DOMException}}.
  2. If [=this=].{{TCPSocket/[[closedPromise]]}} is already resolved, return [=this=].{{TCPSocket/[[closedPromise]]}}.
  3. Let |cancelPromise:Promise| be the result of invoking [=ReadableStream/cancel=] on [=this=].{{TCPSocket/[[readable]]}}.
  4. Mark |cancelPromise| as handled.
  5. Let |abortPromise:Promise| be the result of invoking [=WritableStream/abort=] on [=this=].{{TCPSocket/[[writable]]}}.
  6. Mark |abortPromise| as handled.
  7. [=Resolve=] [=this=].{{TCPSocket/[[closedPromise]]}} with `undefined`.
  8. Return [=this=].{{TCPSocket/closed}}.

Integrations

Permissions Policy

This specification defines a feature that controls whether {{TCPSocket}} and {{UDPSocket}} classes may be created.

The feature name for this feature is "direct-sockets"`.

The [=default allowlist=] for this feature is `'self'`.

A document’s permission policy determines whether a `new TCPSocket(...)` or `new UDPSocket(...)` call rejects with a {{"NotAllowedError"}} {{DOMException}}.
